토스증권 Open API 기능 정리

기준일 2026-06-25 · Base URL https://openapi.tossinvest.com · 본 문서는 company-map 연동 과정에서 실제 호출로 검증한 내용을 정리한 것입니다.

핵심 성격 — 국내(KRX/NXT) + 미국 주식의 시세 · 종목정보 · 환율/장운영 · 계좌/보유 · 주문을 다루는 REST API입니다. WebSocket 미지원, 모든 응답 JSON, 금액·수량은 문자열, 타임존은 KST.

1. 인증 (Auth)

  • OAuth2 Client Credentials: 토스증권 WTS > Open API에서 client_id/client_secret 발급
  • POST /oauth2/token (form-urlencoded, grant_type=client_credentials) → access_token(약 24시간), token_type, expires_in
  • 모든 요청 헤더: Authorization: Bearer {access_token}
  • 계좌·자산·주문 API는 X-Tossinvest-Account: {accountSeq} 헤더 추가 필수
⚠️ 클라이언트당 활성 토큰 1개만 유지됩니다. 새 토큰을 발급하면 이전 토큰이 즉시 무효화되므로, 동시 다발 재발급은 서로를 무효화합니다(토큰 발급을 single-flight로 묶어야 안전).

2. 제공 기능 (카테고리별 엔드포인트)

시세 (Market Data) — 토큰만 필요

기능메서드 · 경로주요 파라미터 / 응답
현재가GET /api/v1/pricessymbols=(콤마, 최대 200) → lastPrice, currency, timestamp
호가GET /api/v1/orderbooksymbol= → asks[], bids[] (각 10단계, price/volume)
최근 체결GET /api/v1/tradessymbol= → 체결 목록 price, volume, timestamp, currency
상/하한가GET /api/v1/price-limits가격 제한
캔들GET /api/v1/candlessymbol=, interval=1m|1d, count=1~200, before=, adjusted= → open/high/low/closePrice, volume (최신순)

종목 정보 (Stock Info) — 토큰만 필요

기능메서드 · 경로주요 응답
종목 기본정보GET /api/v1/stockssymbols= → name, englishName, isinCode, market, securityType, currency, listDate, sharesOutstanding, koreanMarketDetail
매수 유의사항GET /api/v1/stocks/{symbol}/warnings투자유의/경고 정보

시장 정보 (Market Info) — 토큰만 필요

기능메서드 · 경로주요 응답
환율GET /api/v1/exchange-ratebaseCurrency=&quoteCurrency= (예: USD→KRW) → rate, midRate
국내 장 운영정보GET /api/v1/market-calendar/KR전·당·익영업일의 preMarket / regularMarket / afterMarket 시각
미국 장 운영정보GET /api/v1/market-calendar/USdayMarket / preMarket / regularMarket / afterMarket 시각

계좌 · 자산 (Account / Asset) — 계좌 헤더 필요

기능메서드 · 경로주요 응답
계좌 목록GET /api/v1/accountsaccountNo, accountSeq, accountType (BROKERAGE / OVERSEAS_DERIVATIVES / PENSION_SAVINGS / RESHORING_INVESTMENT)
보유 주식GET /api/v1/holdings요약(총매입·평가금액·평가손익·당일손익, 통화별) + 종목별 quantity, lastPrice, averagePurchasePrice, marketValue, profitLoss, dailyProfitLoss, cost

주문 (Order / History / Info) — 계좌 헤더 필요

기능메서드 · 경로주요 파라미터 / 응답
주문 생성POST /api/v1/orderssymbol, side(BUY/SELL), orderType(LIMIT/MARKET), quantity|orderAmount, price(LIMIT시), timeInForce(DAY/CLS), clientOrderId → orderId
주문 정정POST /api/v1/orders/{orderId}/modify미체결 주문의 가격·수량 정정
주문 취소POST /api/v1/orders/{orderId}/cancel미체결 주문 취소
주문 목록GET /api/v1/ordersstatus=OPEN|CLOSED
주문 상세GET /api/v1/orders/{orderId}개별 주문 상태
매수 가능 금액GET /api/v1/buying-powercurrency=KRW|USD → cashBuyingPower
매도 가능 수량GET /api/v1/sellable-quantitysymbol= → sellableQuantity
매매 수수료GET /api/v1/commissions시장별 commissionRate (현재 국내 0% · 미국 0.1%)

3. 속도 제한 (Rate Limit · 초당 요청 수)

그룹한도/초피크(09:00~09:10)그룹한도/초
AUTH5—MARKET_DATA10
ACCOUNT1—MARKET_DATA_CHART5
ASSET5—ORDER6 → 3
STOCK5—ORDER_HISTORY5
MARKET_INFO3—ORDER_INFO6 → 3

응답 헤더 X-RateLimit-Limit / Remaining / Reset, 초과 시 429 + Retry-After. ACCOUNT(초당 1회)이 가장 빡빡 → 계좌 목록은 캐싱 권장.

4. 주요 에러 코드

HTTP · code의미 / 대응
400 invalid-request파라미터/형식 오류
400 confirm-high-value-required1억원 이상 주문 → confirmHighValueOrder: true 추가
400 account-header-requiredX-Tossinvest-Account 헤더 누락
401 invalid-token / expired-token토큰 재발급 (동시 재발급 주의)
422 insufficient-buying-power매수가능금액 부족
422 order-hours-closed주문 가능 시간 외
422 price-out-of-range상/하한가 초과
429 rate-limit-exceededRetry-After 대기 후 재시도
404 stock-not-found / order-not-found, 409 already-filled, 403 forbidden, 500 internal-error각 상황별 처리

에러 형식: { error: { requestId, code, message, data } } — CS 문의 시 requestId 첨부.

5. 한계 (없는 기능)

  • 종목명 검색 없음 — /stocks는 symbol(티커)로만 조회. 이름→심볼 변환은 외부(예: 네이버 자동완성)로 보완해야 함.
  • 실시간 스트리밍(WebSocket) 없음 — 시세는 폴링.
  • 전일종가 전용 필드 없음 — 현재가에 포함되지 않아 캔들/당일손익률로 산출해야 함.
  • 시간외(프리/애프터마켓) 별도 가격 필드 없음 — 현재가가 최신 체결(시간외 포함)을 반영. "시장외 여부"는 장 운영정보로 판정.
