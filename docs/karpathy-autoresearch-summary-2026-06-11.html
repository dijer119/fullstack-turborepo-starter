karpathy/autoresearch — 기능 정리

저장소: github.com/karpathy/autoresearch · 라이선스: MIT · 정리일: 2026-06-11

1. 한 줄 요약

AI 에이전트(Claude 등)가 단일 GPU에서 LLM 훈련 코드를 스스로 수정하고 5분짜리 실험을 반복하며 검증 손실을 자동으로 개선해 나가는 "자율 연구" 환경. 사람이 자는 동안 에이전트가 하룻밤에 ~100회 실험을 돌리는 구조다.

2. 핵심 아이디어

  • 간소화된 nanochat(단일 GPU GPT 훈련) 코드를 베이스로, 에이전트가 train.py를 수정 → 고정 5분 시간 예산으로 훈련 → val_bpb(검증 bits/byte, 낮을수록 좋음) 측정 → 개선되면 유지, 아니면 폐기하는 루프를 자동 반복.
  • 시간당 약 12회, 하룻밤 약 100회 실험이 가능한 빠른 피드백 루프.
  • vocab 크기와 무관하게 공정 비교가 되도록 평가 메트릭을 bits/byte로 설계.

3. 파일 구조 (역할 분담이 핵심 설계)

파일수정 주체역할
prepare.py고정 (수정 금지)데이터 다운로드, 토크나이저 학습, 평가 유틸리티 (~2분 1회 실행)
train.py에이전트가 편집GPT 모델 정의, 옵티마이저(Muon + AdamW), 훈련 루프 — 하이퍼파라미터·아키텍처·배치 크기 등 전부 수정 가능
program.md사람이 편집에이전트에게 주는 지시사항·컨텍스트 ("연구 프로그램")

에이전트가 한 파일(train.py)만 수정하게 범위를 제한해 diff 검토와 실험 관리가 쉽도록 한 것이 의도된 설계.

4. 사용법

# 설치 (uv 기반)
curl -LsSf https://astral.sh/uv/install.sh | sh
uv sync

# 초기 1회: 데이터 + 토크나이저 준비 (~2분)
uv run prepare.py

# 단일 실험: 5분 훈련
uv run train.py

# 자율 모드: Claude 등 에이전트에게
#   "program.md를 보고 새 실험을 시작하자" 라고 지시

5. 기술 스택·요구사항

항목내용
언어/라이브러리Python 3.10+, PyTorch (Python 83.4% + Jupyter 16.6%)
패키지 관리uv
하드웨어NVIDIA GPU 필수 (H100에서 테스트). CPU/MPS 미지원 — 단, MacOS·Windows·AMD 지원 포크가 "Notable forks"로 README에 나열됨
평가 메트릭val_bpb (validation bits per byte, 낮을수록 좋음)

6. 저사양 환경 권장 설정 (README)

  • 엔트로피 낮은 데이터셋(TinyStories 등) 사용
  • vocab_size 축소: 8192 → 4096 → 1024
  • MAX_SEQ_LEN 대폭 축소 (256까지)
  • DEPTH 축소 (8 → 4), 윈도우 패턴 "L" 권장

7. 제약·주의점

  • 고정 시간 예산 방식이라 하드웨어가 다르면 결과를 서로 비교할 수 없음 — 각자 자기 플랫폼에 최적화하는 구조.
  • 5분 예산에 시작·컴파일 시간은 포함되지 않음.
  • 에이전트에게 코드 수정·실행 권한을 주는 구조이므로 신뢰 가능한 샌드박스/환경에서 돌리는 것이 전제.
