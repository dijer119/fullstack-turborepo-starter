무한매수법 V2.2 규칙 정리 (학습 노트)

출처: 라오어 카페 계보 정리 + 나무위키/브런치 등(사용자 제공 분석). 2023년부터 V2.2가 오피셜. 본 문서는 구현 전 학습·확인용이며, 실제 구현은 별도 spec/plan으로 진행. 수치는 3배 레버리지 고위험 전략이라 LIVE 전 원문 재검증 권장.

0. 버전 계보 (핵심 한 줄)

버전매수매도
v10.5회 평단 + 0.5회 시중가+15%(무제한 큰수)전량 평단+10% 1건
v2.0큰수에 평단+5% 고정 상한 도입 + 후반전(진행률50%↑ 평단이하만) + 쿼터손절전량 평단+10% 1건
v2.1v2.0과 동일LOC 세분화 — 전반 2단 / 후반 3단
v2.2 (오피셜)큰수 = 평단+(10−T/2)% 변동식전/후반 구분 폐지 → 2단 통일 (1/4 LOC + 3/4 지정가)

v2.2의 정체성: v2.0/v2.1의 "고정 티어 + T=20 계단(cliff)"을 매수·매도 모두 (10−T/2)%라는 하나의 변동식(경사, ramp)으로 통일. → 매수·매도 둘 다 변경됨(매도 변화가 더 큼).

1. T(회차)와 변동식

  • 원금 40분할, 1회 매수액 = 원금 ÷ 40. 종목 TQQQ 등 3배 레버리지.
  • T = 누적 매수액 ÷ 1회 매수액 (진행률 = T/40).
  • 변동식 값 = (10 − T/2)% (평단 대비). 검증점: T=0 → +10%, T=10 → +5%(= v2.0 큰수와 일치), T=20 → 0%(평단), T=30 → −5%, T=40 → −10%.
  • 전반전 = T < 20, 후반전 = T ≥ 20 (변동식이 0% 이하로 내려가는 지점).

2. 매수 (전부 LOC = LIMIT+CLS)

LOC 매수는 종가 ≤ 지정가일 때 종가로 체결(항상 지정가 이하로 유리).

구간주문
전반전 (T<20)매일 2건: 0.5회분 @ 평단(LOC) + 0.5회분 @ 평단+(10−T/2)%(LOC)
후반전 (T≥20)매일 1건: 1회분 전액 @ 평단+(10−T/2)%(LOC). T≥20이라 이 값 ≤ 0% = 평단 이하

큰수 최종 지정가 = min( 평단+(10−T/2)%, 현재가+15% ) — 급등일에 현재가+15%가 상한으로 작동.

우리 v2.1 구현과의 매수 차이: v2.1 전반 큰수 = 평단×1.05 고정, 후반 = 전액 평단(0%). v2.2는 둘 다 (10−T/2)% 변동(전반은 +5%가 아니라 T에 따라 +10%~+0.5%, 후반은 0%가 아니라 T에 따라 0%~−10%).

3. 매도 (전/후반 구분 폐지 → 2단 통일)

매일 2건:

비중(누적 보유수량)가격유형
1/4 (25%)평단+(10−T/2)%LOC 매도 (종가 ≥ 지정가면 종가로 체결)
3/4 (75%)평단+10% (고정)지정가 매도 (장중 도달 시)

LOC 매도 티어(1/4)가 T에 따라 내려오는 것이 핵심: T=0 → +10%(3/4와 동일), T=10 → +5%, T=20 → 0%(평단), T=30 → −5%. 자금을 많이 쓸수록(T↑) 더 낮은 수익률에서도 1/4을 털어 디리스킹. LOC 종가체결이라 지정가보다 높은 초과수익도 가능(백테스트상 수익률은 v2.0과 비슷하다는 주장).

우리 v2.1 구현과의 매도 차이: v2.1은 전반 2단(25%@+5% LOC / 75%@+10%) · 후반 3단(25%@평단 LOC / 25%@+5% / 50%@+10%). v2.2는 전/후반 무관 2단 통일(1/4 @ (10−T/2)% LOC / 3/4 @ +10%).

4. 쿼터모드 / 쿼터손절 (검증 필요)

  • 쿼터 매도 모드: T가 19~20 사이일 때 1/4을 MOC 매도. 그리고 "쿼터 매도한 날엔 매수하지 않는다"는 예외 규칙(v2.1엔 없음).
  • 쿼터손절 모드: T 39.1~40.0 구간(40회 소진 부근). 전량손절 대신 보유의 1/4을 매도해 시드 재확보 후 계속 진행.

쿼터모드의 정확한 트리거·수량·MOC 처리·"매도한 날 매수 스킵"은 출처가 2차 자료(나무위키/유튜브)라 LIVE 구현 전 원문 재검증 필수. 우리 v2.1은 사용자 결정으로 쿼터손절을 제외했었음.

6. Toss 매핑 (구현 관점)

  • LOC(매수/매도) → LIMIT + timeInForce:"CLS". 지정가 매도 → LIMIT + timeInForce:"DAY". (v2.1에서 검증된 매핑 재사용)
  • MOC(쿼터 매도) → 토스 지원 여부 확인 필요(없으면 종가 근사/시장가 대체 검토).
  • 매도 3/4 지정가 + 1/4 LOC = 동시 2건(v2.1 후반 3건보다 단순). 동시 다중 매도 수용은 여전히 실거래 검증 대상.

7. 우리 코드에 도입 시 변경 범위 (요약)

  • 신규 코어 computeDailyOrdersV22 추가(기존 strategy.ts v1 · strategy-v21.ts는 유지). cycle.version에 "v2.2" 추가, 실행기 분기.
  • 핵심은 (10−T/2)% 변동식 하나로 매수 큰수·매도 1/4 LOC를 계산. 큰수는 min(…, 현재가+15%) 캡. 정수 반올림(합=보유수량) 규칙은 v2.1과 동일.
  • 쿼터모드는 별도 결정 필요(포함/제외).

8. 미확인 / 검증 필요

  1. 쿼터모드(T 19~20 MOC 매도 + "매도일 매수 스킵")·쿼터손절(T 39.1~40) 정확 규칙.
  2. 매도 1/4·3/4의 정수 반올림 정확 방식(누적 수량 기준).
  3. 매수 전반전 0.5/0.5 분할이 v2.2에서도 유지되는지(2차 자료 기준 유지로 보이나 원문 확인).
  4. 종목별 목표 차등(SOXL 등)·현재가+15% 캡의 정확 적용.
  5. (참고) v3.0은 별개: TQQQ 매도점 +10%→+15%, 40→20분할. v2.2 범위 아님.
