ETF 시가총액 표시 — 설계 명세

작성일: 2026-06-11 · 대상 앱: apps/company-map · 페이지: /stocks/etf?code=<ETF코드>

1. 목적

관심 ETF 상세 화면 헤더에 해당 ETF의 시가총액을 한국식 단위("1,128억")로 표시한다. 현재 값 표시만 — 이력 UI는 범위 외 (단, 일별 스냅샷에 저장되므로 이력 데이터는 자동 축적).

2. 데이터 소스 (라이브 확인 완료)

  • 이미 일별 수집 중인 Naver 모바일 etfAnalysis 응답에 marketValue 필드가 한국어 단위 문자열로 존재 — 예: "1,128억" (476850에서 2026-06-11 확인). 새 외부 API 불필요.
  • 참고: ETF는 일반 주식과 달리 integration API의 totalInfos에 marketValue가 없다 — 기존 fetchMarketValue()(lib/stocks/price-history.ts)는 ETF에 재사용 불가.
  • 파싱·포맷 유틸 재사용: parseKoreanMarketValue()(lib/stocks/parse-market-value.ts, "1,128억" → BigInt 원), formatMarcap()(lib/format-marcap.ts, BigInt/number → "1,128억").

3. 아키텍처 (A안: 스냅샷 저장)

구성요소변경
prisma/schema.prismaEtfPdfSnapshot에 marketValue BigInt? @map("market_value") 추가 (nullable — 과거 스냅샷은 null). 마이그레이션 1회.
src/lib/etf/naver.tsparseNaverEtfAnalysis() 반환에 marketValue: bigint | null 추가 — parseKoreanMarketValue(json.marketValue) 위임. 단위 테스트 추가.
worker/fetch-naver-etf.tsfetchNaverEtf() 반환에 marketValue 전달.
worker/etf-snapshot.ts① 스냅샷 create 시 marketValue 저장. ② 당일 스냅샷이 이미 있고 marketValue가 null이면 그 필드만 update 후 "saved" 대신 "updated" 반환 (없으면 기존 skip 로직 때문에 다음 영업일까지 값이 안 들어옴).
src/actions/etf.tsgetEtfDetail()이 최신 스냅샷의 marketValue를 EtfDetailView.marketValue: number | null로 추가. BigInt → Number() 변환 (server→client 직렬화에 BigInt 불가; 시총 규모 ≪ 2^53이라 정밀도 안전).
src/app/stocks/etf/EtfManager.tsx기준일 줄에 · 시가총액 1,128억 병기 — formatMarcap(detail.marketValue). null이면 해당 조각 생략.

4. 동작 상세

  • snapshotEtf 상태값: 기존 "saved" | "skipped" | "failed"에 "updated" 추가 (당일 스냅샷의 marketValue 보강 시). 로그 출력 외 소비자 없음 — 호환성 영향 없음.
  • marketValue 파싱 실패(필드 없음/형식 불일치) 시 null 저장 — 스냅샷 저장 자체는 실패하지 않는다.
  • UI: detail.marketValue == null이면 "시가총액" 조각을 렌더하지 않음 (— 표시도 하지 않음; 과거 스냅샷만 있는 ETF에서 불필요한 빈 표기 방지).
