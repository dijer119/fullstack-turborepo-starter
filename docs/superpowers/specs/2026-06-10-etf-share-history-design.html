ETF 구성종목 주식수 변경 이력 — 설계 명세

작성일: 2026-06-10 · 대상 앱: apps/company-map · 페이지: /stocks/etf?code=<ETF코드>

1. 목적

관심 ETF 상세 화면에서 각 구성종목의 보유 주식수가 날짜별로 어떻게 변해왔는지를 한눈에 보고, 종목별 추이를 차트로 확인한다. 현재 화면은 최신 vs 직전 스냅샷 2개만 비교하므로 그 이전의 흐름(점진적 매집/매도 패턴)을 볼 수 없다.

2. 전제 (변경 없음)

  • 일별 스냅샷은 이미 EtfPdfSnapshot + EtfPdfHolding에 누적되고 있다 (worker etf-pdf-loop + 수동 갱신). 스키마·수집 로직 변경 없음.
  • 데이터 소스는 Naver 모바일 ETF API의 Top10 구성종목뿐이다. 따라서 어떤 종목이 Top10 밖으로 밀려난 날짜의 주식수는 알 수 없다 → 이력에 공백이 생기며 —로 정직하게 노출한다.

4. 데이터 계약

// src/lib/etf/history.ts
export interface ShareHistoryRow {
  constituentCode: string;
  constituentName: string;
  latestWeight: number | null;   // 정렬 기준 (최신 스냅샷 비중)
  inLatest: boolean;             // 최신 Top10 포함 여부
  cells: (ShareHistoryCell | null)[]; // dates와 같은 길이, 해당 날짜 Top10에 없으면 null
}

export interface ShareHistoryCell {
  shares: number | null;
  delta: number | null; // 직전 "데이터가 있는" 셀 대비 증감. 첫 등장이면 null
}

export interface ShareHistory {
  dates: string[];               // trdDd 오름차순 (좌→우 시간순)
  rows: ShareHistoryRow[];       // 정렬: inLatest 우선, latestWeight 내림차순, 이탈 종목은 하단
}

export function buildShareHistory(
  snapshots: { trdDd: string; holdings: Holding[] }[], // 순서 무관, 내부에서 trdDd 정렬
): ShareHistory;
  • delta는 직전 데이터가 있는 셀 대비로 계산한다 (중간 공백을 건너뜀). Top10 재진입 시 마지막 관측치와 비교하는 것이 "변경 이력" 목적에 부합.
  • 종목 식별은 constituentCode 기준. 코드가 빈 문자열인 행(파서가 이름만 준 경우)은 constituentName으로 fallback 식별.
  • 액션 시그니처: getEtfShareHistory(code: string, limit = 30): Promise<ShareHistory | null> — 워치 없으면 null, 스냅샷 0개면 { dates: [], rows: [] }.

5. UI 설계

5.1 매트릭스 표

기존 비교표 아래에 새 섹션 "주식수 변경 이력"을 추가한다.

종목06-0506-0806-0906-10
삼성전자120,000120,000125,000 ▲5,000125,000
SK하이닉스45,00043,000 ▼2,00043,00048,000 ▲5,000
현대차——12,000 신규12,000
  • 열 = 최근 30개 스냅샷의 trdDd, 좌→우 시간순. overflow-x-auto 가로 스크롤, 종목명 첫 열은 sticky left-0 고정.
  • 셀 = shares.toLocaleString(). delta가 있으면 증감 수치를 작게 병기하고 기존 deltaColor 컨벤션(증가=초록, 감소=빨강) 적용. 첫 등장 셀은 "신규" 표기. 데이터 없는 날짜는 — (회색).
  • 행 정렬: 최신 Top10 포함 종목을 최신 비중 내림차순으로 먼저, 이탈 종목(최신 미포함)은 하단에 회색 톤으로.
  • 스냅샷이 1개 이하면 "이력이 쌓이는 중입니다 (스냅샷 N개)" 안내만 표시.

5.2 종목별 라인 차트

  • 행 클릭 → 표 아래에 해당 종목의 주식수 추이 Recharts LineChart 확장 (X=trdDd, Y=shares). 다시 클릭하면 접힘. 한 번에 1종목만.
  • 데이터 없는 날짜는 connectNulls로 선을 이어 추세를 보이게 하되, 점은 관측치에만 찍는다.
  • recharts ^3.8.1 기설치 — 의존성 추가 없음. 차트 컴포넌트는 client component인 ShareHistorySection 내부에 둔다.

6. 에러 처리

  • 이력 조회 실패가 페이지 전체를 깨지 않도록 page.tsx에서 getEtfShareHistory 실패 시 null로 처리하고 섹션만 숨긴다 (기존 detail 영역은 영향 없음).
  • buildShareHistory는 입력 스냅샷이 비어 있어도 throw하지 않고 빈 결과를 반환한다.

7. 테스트 계획

케이스검증 내용

8. 범위 제외 (YAGNI)

  • ETF 자체 상장주식수(CU 설정/환매) 추적 — 별도 작업으로 분리.
  • 비중(weight) 이력 매트릭스 — 이번엔 주식수만. 구조상 추후 확장 용이.
  • 기간 선택 UI — 최근 30 스냅샷 고정. 필요해지면 추가.
  • 스키마·worker 수집 로직 변경 — 없음.
알려진 한계: Naver Top10 기반 수집이므로 Top10 밖 기간은 이력 공백(—)으로 표시된다. 또한 스냅샷 수집이 시작된 날짜 이전의 이력은 존재하지 않는다.
