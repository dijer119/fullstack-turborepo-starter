# compound-cleanup 스킬 설계

## 배경

이 환경은 "학습을 축적"하는 Compound Engineering 방식을 이미 실천 중이다.
하지만 축적 메커니즘만 있고 **가지치기(pruning)·검증 메커니즘이 없다.**
복리는 좋은 것에만 작동하지 않는다 — 잘못된 패턴, 코드와 어긋난 규칙,
이미 해결된 주의사항도 함께 복리로 쌓여 이후 모든 사이클에 오염을 전파한다.

`compound-cleanup` 스킬은 이 약점을 보완하는 **읽기 전용 진단 도구**다.
축적된 컨텍스트를 검토해 정리 후보를 리포트로 제시할 뿐, 아무것도 수정하지 않는다.
실제 정리는 사용자가 리포트를 보고 직접 판단·수행한다.

## 결정 사항 (확정)

| 항목 | 결정 | 이유 |
|------|------|------|
| 검토 범위 | ① + ② + ③ 전체 | 세 저장소 모두 stale·중복·모순의 발생원 |
| 동작 방식 | **읽기 전용 보고** | 손수 큐레이션한 ②③의 자동 삭제 위험 회피, surgical-changes 원칙 부합 |
| 검증 차원 | A·B·C (D 제외) | D(노후)는 판단 개입이 커 오탐 多 |
| 검증 깊이 | 명시된 경로·심볼만 핀포인트 | 코드 전체 grep은 과중, 비용 대비 가치 낮음 |
| 출력 형식 | `docs/` 하위 HTML 리포트 | 글로벌 CLAUDE.md 규칙 #5 (구조화 검토 결과는 HTML) |
| 설치 위치 | 개인 스킬 `~/.claude/skills/` | 글로벌 CLAUDE.md를 검토 대상에 포함하므로 |

## 검토 대상 (읽기만, 수정 없음)

| # | 저장소 | 위치 |
|---|--------|------|
| ① | Compound Learning 섹션 + 글로벌 가이드 | `~/.claude/CLAUDE.md` |
| ② | 프로젝트 가이드 | 저장소 루트 `CLAUDE.md` |
| ③ | 파일 기반 메모리 | `<project>/.../memory/*.md` + `MEMORY.md` |
| - | 검증용 코드 | 프로젝트 소스 (`grep`/`ls` 읽기 전용) |

## 검증 차원

### A. 끊어진 참조 (Dangling references) — ★★★ 객관적
①②③이 언급한 파일·디렉토리·함수·심볼·명령어가 실제로 존재하는지 확인.
- 대상: CLAUDE.md/메모리 본문에 **명시적으로 적힌 경로·심볼만** 추출
- 방법: 경로는 `ls`, 심볼은 핀포인트 `grep`
- 결과: 존재하지 않으면 정리 후보

### B. 중복 (Duplicates) — ★★
같은 사실이 ①②③ 여러 저장소에 중복 기록됨.
- 예: 동일 규약이 프로젝트 CLAUDE.md와 memory 양쪽에 존재
- 결과: 중복 위치 쌍을 리포트, 단일 source of truth 권장

### C. 모순 (Contradictions) — ★★
두 항목이 서로 충돌.
- 예: "Pages Router 사용" ↔ "App Router로 전환됨"
- 결과: 충돌 항목 쌍과 충돌 지점 리포트

> **제외:** D(노후) — 이미 해결된 주의사항/오래된 사이클 메모. 판단 개입이 커 오탐이 많아 이번 범위에서 제외.

## 출력

`docs/` 하위에 자체 완결형 HTML 리포트(인라인 CSS, 외부 의존성 0) 생성.

각 정리 후보 행:

| 컬럼 | 내용 |
|------|------|
| 위치 | 저장소(①②③) + 파일 + 라인/섹션 |
| 분류 | A / B / C |
| 내용 | 해당 항목 원문 발췌 |
| 근거 | 왜 정리 후보인지 (예: "참조한 `foo.ts` 없음") |
| 권장 조치 | 삭제 / 갱신 / 통합 등 |

채팅에는 리포트 절대 경로만 알린다. 본문을 채팅에 늘어놓지 않는다.

## 비목표 (Non-goals)

- 자동 삭제·수정 (읽기 전용 고수)
- 코드 전체 스캔 (명시된 심볼만 핀포인트)
- 차원 D(노후) 탐지
- 새 컨텍스트 축적 (이건 기존 메모리/Compound Learning 메커니즘의 역할)

## 성공 기준

- 스킬 실행 시 ①②③을 모두 읽고 A·B·C 후보를 빠짐없이 분류
- 끊어진 참조(A)는 실제 `ls`/`grep` 결과에 근거 (추측 금지)
- HTML 리포트가 브라우저에서 바로 읽힘 (인라인 스타일)
- 어떤 저장소 파일도 수정되지 않음
