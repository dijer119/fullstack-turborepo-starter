VIP 펀드 추적 기능 설계

날짜: 2026-06-28 · 앱: company-map · 대상: VIP한국형가치투자증권자투자신탁[주식]A-e클래스 (운용사: 브이아이피자산운용, KB amak_stnd_cd=K553W5E17302)

1. 목표

KB증권 펀드 상세 페이지의 데이터를 ETF 추적 기능과 유사하게 주기적으로 스냅샷·추적한다. 단일 펀드(하드코딩)이며, 추적 대상은 종목별 보유 TOP10과 기준가(NAV) 추이 두 가지다.

성공 기준:

  • 종목별 TOP10 보유종목이 바뀔 때마다 새 스냅샷이 쌓이고, 신규/이탈/비중변동 diff를 화면에서 확인할 수 있다.
  • NAV 주간 시계열이 차트로 표시된다.
  • /stocks/funds 별도 화면 + 사이드바 메뉴로 접근 가능하다.

2. 데이터 소스 (검증 완료)

브라우저 불필요. 서버사이드 Node fetch + EUC-KR 디코드로 전부 수집 가능함을 확인했다.

데이터요청형식
펀드명·운용사·종목별 비중 TOP10GET go.able?linkcd=s010100030000&amak_stnd_cd=K553W5E17302EUC-KR HTML
기준가(NAV) 주간 시계열GET chart/data/MOBI_02_0039_JSON.jsp?dd=1&amak_stnd_cd=...JSON {data:[{date,fnd}]}

HTML 내 "주식종목별비중TOP10" caption 표에서 <td >종목명</td><td >비중</td> 행을 파싱한다(종목별 표는 셀이 <td > 공백 포함이라 정규식 주의). NAV의 fnd는 기준가(원), date는 YYYYMMDD(KST).

업종별·국가별·자산유형별 비중은 이번 범위에서 제외(요청 범위: 종목별 + NAV).

3. 스냅샷 dedup 전략 (ETF와 다른 핵심 결정)

ETF는 KRX가 매일 명확한 기준일 PDF를 제공해 trdDd로 dedup하지만, 펀드 보유종목은 운용보고서 기준이라 드물게(월 단위) 변하고 페이지에 일별 기준일이 없다. 따라서 내용 기반 dedup을 쓴다.

  • 새로 가져온 TOP10(종목명+비중 집합)을 최신 스냅샷과 비교.
  • 동일하면 skip(새 행 생성 안 함). 달라졌을 때만 그날 날짜(KST YYYYMMDD)로 새 스냅샷 생성.
  • @@unique([fundCode, trdDd])로 같은 날 재실행 시 중복 방지.

결과적으로 보유 이력은 실제 구성 변동이 있을 때만 늘어난다.

4. 데이터 모델 (company-map SQLite, 별도 테이블)

ETF 테이블 재사용 대신 별도 모델을 둔다(코드 형식·NAV 차이, ETF 목록 UI 오염 방지). 단일 펀드라 등록 테이블은 없고 펀드 코드/이름은 상수로 둔다.

model FundSnapshot {
  id        String        @id @default(uuid())
  fundCode  String        @map("fund_code")   // amak_stnd_cd
  trdDd     String        @map("trd_dd")       // YYYYMMDD (생성일)
  nav       Float?                             // 스냅샷 시점 최신 기준가(참고)
  fetchedAt DateTime      @default(now()) @map("fetched_at")
  holdings  FundHolding[]
  @@unique([fundCode, trdDd])
  @@map("fund_snapshot")
}

model FundHolding {
  id         String       @id @default(uuid())
  snapshotId String       @map("snapshot_id")
  name       String
  weight     Float?
  rank       Int
  snapshot   FundSnapshot @relation(fields: [snapshotId], references: [id], onDelete: Cascade)
  @@map("fund_holding")
}

model FundNav {
  id       String @id @default(uuid())
  fundCode String @map("fund_code")
  date     String                            // YYYYMMDD
  nav      Float
  @@unique([fundCode, date])
  @@map("fund_nav")
}

6. diff 재사용

ETF src/lib/etf/diff.ts의 diffHoldings(latest, prev)는 식별자로 constituentCode || constituentName을 쓰므로 name 기반 펀드 보유에도 그대로 동작한다. 펀드 보유를 {constituentCode:"", constituentName:name, weight, shares:null, amount:null} 형태로 매핑해 재사용한다. (펀드엔 주식수가 없으므로 sharesDelta 컬럼은 UI에서 숨김.)

7. 수동 새로고침

ETF의 무거운 spawn-script + refresh_state 머신을 쓰지 않고, 서버액션 refreshFund()에서 인라인으로 fetch→snapshotFund()→revalidatePath. 단일 빠른 fetch라 충분.

8. 합산 보기 (선택적, stretch)

/stocks/etf/combined에 ?fund=1이 있으면 펀드 최신 보유 TOP10을 별도 컬럼으로 병기한다. ETF↔펀드는 성격이 달라 비중 단순 합산은 하지 않고 컬럼만 추가한다. 본 기능들이 끝난 뒤 진행.
