무한매수법 V2.1 도입 설계

날짜: 2026-07-04 · 앱: company-map · 규칙 출처: docs/superpowers/specs/2026-07-04-infinite-buy-v2.1-reference.html

1. 목표

기존 무한매수 자동매매(v1)를 그대로 유지하면서, 라오어 V2.1을 사이클별로 선택 가능한 별도 전략 버전으로 추가한다. 매수·매도 로직만 V2.1을 따르고, 손절/원금소진 후 처리는 v1과 동일(쿼터손절 제외).

성공 기준:

  • 사이클 등록 시 v1/v2.1 선택 → worker가 버전에 맞는 일일 주문을 생성.
  • V2.1 전략 코어가 순수 함수로 분리되어 Vitest로 검증(전반/후반, 매수 2종, 매도 2·3분할, 정수 반올림, 큰수 상한).
  • 기존 v1 사이클·테스트·데이터는 회귀 없이 동작.

2. 확정 사항

  • 전반/후반 = 회차 기준: round < splits/2 전반, round ≥ splits/2 후반 (40분할이면 20).
  • 버전 병행: InfiniteBuyCycle.version 컬럼 추가("v1" 기본 / "v2.1").
  • 큰수매수 상한 = 평단+5% (LOC 주문가 고정으로 내재 보장 + assert).
  • 정수 반올림: LIMIT 주문은 정수 수량만 → 분할 합계 = 보유수량 보장.
  • 손절/원금소진 후 = v1과 동일. 쿼터손절·가상증액 제외.
  • 동시 다중 매도(후반 3건)는 구현 후 소액 실거래로 검증.

3. 전략 코어 (순수 함수, 신규)

src/lib/infinite-buy/strategy-v21.ts — computeDailyOrdersV21(state). 기존 strategy.ts(v1)는 손대지 않는다. 반환 타입은 v1의 DailyPlan/IntendedOrder 재사용.

3.1 매수 (전부 LOC = LIMIT+CLS)

const daily = principalUsd / splits;           // 1회치
const isSecondHalf = round >= splits / 2;       // 후반전
const capBig = avgPrice * 1.05;                 // 큰수 상한(평단+5%)

if (round === 0) {
  // 최초: 시장가 1회치 (first_market)
  qty = floor(daily / currentPrice); // ≥1이면 BUY MARKET DAY
} else if (!isSecondHalf) {
  // 전반: 0.5회치 평단 LOC + 0.5회치 평단+5% LOC
  half = daily / 2;
  loc_avg:  qty = floor(half / avgPrice)  @ avgPrice        (LIMIT CLS)
  loc_big:  qty = floor(half / capBig)    @ capBig          (LIMIT CLS)  // 주문가 = 평단×1.05 고정
} else {
  // 후반: 1회치 전액 평단 LOC
  loc_avg:  qty = floor(daily / avgPrice) @ avgPrice        (LIMIT CLS)
}

큰수매수 주문가는 항상 평단×1.05(현재가 기반 아님) → 평단+5% 초과 매수 불가.

3.2 매도 (보유 > 0일 때, 전체 보유수량 기준 분할)

LOC = LIMIT+CLS, 지정가 = LIMIT+DAY. 정수 반올림: 작은 구간 floor, 나머지를 +10% 구간에.

구간분할
전반전25% LOC @ 평단×1.05(+5%) · 나머지(≈75%) 지정가 @ 평단×1.10(+10%)
후반전25% LOC @ 평단(0%) · 25% 지정가 @ 평단×1.05(+5%) · 나머지(≈50%) 지정가 @ 평단×1.10(+10%)
// 전반: q5 = floor(N*0.25); q10 = N - q5;
//   sell_loc_5 (LIMIT CLS @ avg*1.05, qty q5)  — q5≥1일 때만
//   sell_lim_10 (LIMIT DAY @ avg*1.10, qty q10)
// 후반: q0 = floor(N*0.25); q5 = floor(N*0.25); q10 = N - q0 - q5;
//   sell_loc_0 (LIMIT CLS @ avg,      qty q0)
//   sell_lim_5 (LIMIT DAY @ avg*1.05, qty q5)
//   sell_lim_10 (LIMIT DAY @ avg*1.10, qty q10)
// 각 구간 qty 0이면 생략. 합계는 항상 N.

매도 kind 신규: sell_loc_5·sell_lim_10·sell_loc_0·sell_lim_5. (v1의 target_sell/reset_sell은 유지; 체결 동기화·이력은 side=SELL 전체를 대상으로 하므로 신규 kind도 자동 포함.)

3.3 원금소진 후 / 손절

v1과 동일: round ≥ splits에서 보유 0이면 리셋, pnlPct ≤ -lossCut이면 시장가 전량 손절(reset_sell) 후 리셋, 그 외엔 위 매도 분할 유지. round 진행/리셋 규칙 동일.

4. 실행기 분기

src/lib/infinite-buy/run.ts의 CycleConfig에 version: "v1" | "v2.1" 추가. runCycle이 버전 보고 computeDailyOrders(v1) 또는 computeDailyOrdersV21 호출. 그 외 로직(가용금액 체크, dryRun/killed, 제출, 로깅, round/리셋 갱신, assertSubmittable)은 전부 공유·불변.

매수 가용금액 체크·avgCost 기록·체결 동기화 등 기존 안전장치는 그대로 적용된다.

5. 데이터 모델

model InfiniteBuyCycle {
  ...
  version String @default("v1")   // "v1" | "v2.1"
  ...
}

주문 모델(InfiniteBuyOrder)은 변경 없음(kind는 String이라 신규 값 그대로 저장).

6. UI

  • 등록 폼에 버전 선택(v1 / v2.1) 추가. 기본 v1.
  • 사이클 카드에 버전 배지 표시(예: v2.1).
  • 주문 이력·매도 체결 이력·가격 차트·USD 잔고 등은 기존 그대로 재사용.

8. 검증 계획

  • 코어 Vitest: 1회차 시장가, 전반 매수 2종·큰수 주문가=평단×1.05, 후반 매수 전액 평단, 전반 매도 25/75, 후반 매도 25/25/50, 정수 반올림 합계=N, 보유 1~2주 degenerate, 큰수 상한(현재가 높아도 주문가 평단×1.05).
  • 회귀: 기존 v1 테스트 15개 유지. tsc 0.
  • 실행기: dryRun으로 v2.1 사이클 1회 실행 → simulated 주문이 규칙대로 로깅되는지.
  • UI: 버전 선택·배지 렌더. (스키마 변경 → dev 서버 재시작 필요.)
  • LIVE 검증(사용자): 소액 v2.1 사이클로 후반전 3건 동시 매도가 Toss에 접수되는지 확인.

9. 범위 밖 / 주의

  • 쿼터손절·가상증액 제외.
  • 동시 다중 매도 수용은 문서 미명시 → 실거래 검증 전제. 제출 실패는 건별 로깅(하나 실패가 나머지 막지 않음).
  • Prisma 스키마 변경 후 dev 서버·worker 재시작 필요. worker는 정확히 1개만.
