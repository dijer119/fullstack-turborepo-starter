종목 상세 퀀트 대시보드 — 설계 명세

작성일: 2026-06-10 · 대상 앱: apps/company-map (Next.js 16 App Router · SQLite · Vitest) · 참고: 이미지 분석 문서

1. 목표

종목 상세 페이지(/stocks/[code]) 최상단에 Value Sheet 스타일 퀀트 대시보드 섹션을 추가한다. 기존 보유 데이터를 최대한 재사용하고, 신규 외부 의존성은 한국은행 ECOS API(국채금리)와 네이버 일별 시세 1년 범위 조회(52주 고저) 두 가지로 한정한다.

두 신규 조회는 Promise.all로 기존 조회와 병렬 실행한다. Suspense 스트리밍(B안)은 채택하지 않으며, 필요 시 후속 개선으로 전환한다.

5. 데이터 계층

5.1 신규 Prisma 모델 (마이그레이션 1회)

model Week52Price {
  code      String   @id
  high      Float
  low       Float
  asOfDate  DateTime @map("as_of_date")   // 시세 마지막 일자
  fetchedAt DateTime @default(now()) @map("fetched_at")
  master    StockMaster @relation(fields: [code], references: [code], onDelete: Cascade)
  @@map("week52_prices")
}

model MarketRate {
  kind      String   @id              // 예: "treasury_10y"
  ratePct   Float    @map("rate_pct")
  fetchedAt DateTime @default(now()) @map("fetched_at")
  @@map("market_rates")
}

StockMaster에 week52Price Week52Price? 역방향 관계를 추가한다. 기존 RefreshState는 작업 상태 추적용이라 재사용하지 않는다.

5.2 52주 고저 — src/lib/stocks/week52.ts

  • fetchWeek52Range(code): 기존 price-history.ts의 네이버 siseJson 호출 패턴을 재사용해 1년치 일봉을 조회, 고가 컬럼 최대값·저가 컬럼 최소값을 계산
  • getWeek52Price(code): DB 조회 → fetchedAt이 24시간 이내면 그대로 반환, 아니면 fetch 후 upsert. fetch 실패 시 stale 레코드 반환, 그것도 없으면 null

5.3 국채금리 — src/lib/rates/ecos.ts

  • ECOS 통계 API (ecos.bok.or.kr/api/StatisticSearch)에서 국고채 10년 최신 일별 수익률 조회. 기본값은 시장금리(일별) 통계표 817Y002의 국고채 10년 항목으로 하되, 구현 단계에서 ECOS 문서로 항목코드를 실제 호출로 검증 후 확정한다. env(ECOS_TREASURY_STAT_CODE, ECOS_TREASURY_ITEM_CODE)로 재정의 가능
  • env ECOS_API_KEY 필수 — 미설정 시 fetch를 시도하지 않고 캐시(stale 포함)만 사용
  • getTreasuryYield(): MarketRate(kind="treasury_10y") 24h TTL 캐시, 동작은 5.2와 동일한 stale 폴백
  • 응답 파서는 별도 순수 함수 parseEcosYield(json)로 분리 (테스트 대상)

6. UI — src/app/stocks/[code]/QuantDashboard.tsx

  • 위치: StockDetailHeader 바로 아래, 기존 섹션 흐름 위에 추가. SectionNav에 "대시보드" 항목 추가
  • 레이아웃: 카드 2 (기준가·시총) → 게이지 3 (그리드, 모바일 1열) → 세부 지표 카드 4 (배당률, 국채시가배당률, 순이익 YoY, 서준식 지수)
  • GaugeChart.tsx: "use client", recharts PieChart 반원(startAngle 180 → endAngle 0). props: value, min, max, label, color, formatter, ticks. 중앙에 값, 양끝에 min/max 라벨
  • 게이지 색: 52주 위치 = violet, ROE = emerald, PBR = amber. 값이 범위를 벗어나면 클램프해 그리되 실제 값을 표기
  • 서준식 지수 카드: 연복리 기대수익률 % 크게, 아래에 "국채금리 대비 N.N배" 보조 줄. 2.0배 이상이면 emerald 강조
  • 모든 항목은 null이면 "—" 표시. 다크 모드는 기존 dark: Tailwind 패턴 준수, 이미지의 하드코딩 네온 스타일은 차용하지 않음

7. 에러 처리

  • 네이버·ECOS 실패: stale 캐시 → 없으면 해당 카드 "—". 페이지 렌더는 어떤 경우에도 실패하지 않음 (외부 호출은 try/catch + null)
  • 외부 fetch에는 AbortSignal.timeout(10_000) 적용 (blog-collection에서 학습한 패턴)
  • 52주 데이터의 asOfDate를 게이지 옆에 작게 표기해 신선도를 드러냄
