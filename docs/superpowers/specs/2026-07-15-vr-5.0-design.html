VR 5.0 (밸류리밸런싱) company-map 구현 — 설계 스펙

작성일 2026-07-15 · 상태: 사용자 승인 대기 · 선행: 무한매수법 V4.0 (dryRun 전용 자동매매 패턴)

1. 목표와 범위

  • 라오어 VR 5.0(목표가치 V 추종·2주 사이클·LOC 리밸런싱)을 company-map의 dryRun 전용 자동매매로 구현한다. 무한매수법 V4.0과 같은 패턴: worker 폴링 + 토스 주문 계산 + 가상 체결 + 대시보드. LIVE 주문은 게이팅으로 차단(V4.0과 동일).
  • 운용 유형: 적립식 + 거치식 (거치식 = 사이클 적립금 0). 인출식은 범위 외.
  • V 갱신 공식: 기본공식·실력공식 둘 다 구현, 계좌 생성 시 선택 (기본값: 실력공식).
  • 아키텍처: 독립 모듈 — src/lib/vr/ + 새 테이블 + /stocks/vr 페이지 + AppMenu 별도 메뉴 항목. 무한매수법 코드는 건드리지 않고, 재사용은 공용 자원(토스 클라이언트·일봉·dry-fill 판정 로직)만.

범위에서 제외 (후속 과제)

  • 갭 하락 시 시초가 시장가 보정 매수 (라오어 62주차의 수동 대응 — Pool 한도 소진 보정)
  • 인출식 VR (G=20, 음수 현금흐름, Pool 한도 10~25%)
  • 과거 데이터 백테스터
  • LIVE 주문 (dryRun 검증 후 별도 결정)

VrOrder — 주문·체결 이력 (InfiniteBuyOrder 패턴)

accountId, cycleIndex, tradeDate(US), side(BUY|SELL), kind(loc_buy|limit_sell|seed), orderType(LIMIT|MARKET), tif(CLS|DAY), price, quantity, filledQty/filledPrice/filledAt, tossOrderId, stateApplied(V·Pool 반영 멱등 마커), dryRun, status(simulated|simulated_filled|expired|submitted|filled|failed|skipped), error.

VrCycleLog — 사이클 경계 기록 (그래프·감사용)

accountId, cycleIndex, date, vValue(갱신 후), evalAmount(E), pool, contribution, gValue, poolLimitPct, holdingQty. 사이클마다 1행 — 대시보드의 "V vs 평가금" 그래프 원천.

4. 핵심 로직 — src/lib/vr/ (순수 함수 우선)

formula.ts

nextV({v, pool, g, evalAmount, contribution, formula}):
  basic: V₂ = V + Pool/G + 적립금
  skill: V₂ = V + Pool/G + (E − V)/(2√G) + 적립금
band(v, bandPct): { min: v×(1−p/100), max: v×(1+p/100) }

검증 픽스처(62주차 실측): V₁ 15205.76, E 5282.11, Pool 76.73, G 11, 적립 250 → skill 새V = 13966.69, 밴드 = 11871.69 / 16061.69.

pool-limit.ts

poolLimitPct(type, weeksSinceStart):
  accumulate: max(10, 75 − 5 × floor(주차 / 26))   // 75% 시작, 26주마다 −5%p, 최저 10%
  lumpsum:    50                                    // 고정
manual 모드면 poolLimitPct 값 사용

order-table.ts — 매수표·매도표 (62주차 슬라이드에서 역산·검증한 규칙)

  • 매수 (보유 n주, k = n+1, n+2, …): k주째 매수점 가격 = 밴드하단 / (k−1). 주문 = LOC 매수(LIMIT + CLS) 1주. 누적 예상 매수액이 Pool × 사용한도%를 넘는 레벨부터 제외. 검산: 하단 11871.69, 보유 233 → 234주째 = 11871.69/233 = 50.95 ✓, 235주째 = /234 = 50.73 ✓
  • 매도 (k = n−1, n−2, …): k주로 줄이는 매도점 가격 = 밴드상단 / (k+1). 주문 = 지정가 매도(LIMIT + DAY) 1주. 보유수량까지만. 검산: 상단 16061.69, 보유 233 → 232로 = 16061.69/233 = 68.93 ✓, 231로 = /232 = 69.23 ✓
  • 주문 레벨 상한: 매수·매도 각 20레벨 (주문 폭주 방지 — Pool 한도로 대부분 그 이하로 잘림).
  • 평가금이 밴드 안이면 해당 방향 주문이 자연히 트리거되지 않는 가격에 걸린다 — 별도 분기 불필요 (라오어 방식 그대로).

run-vr.ts — 오케스트레이터 (V4 패턴, 순서 불변식: 상태 반영 → 주문 계산)

① sync   : 전일 주문 체결 판정
           dryRun → 전일 일봉으로 judgeDryFill (infinite-buy/dry-fill.ts 재사용)
           LIVE  → 토스 체결 대조 (범위 외, 인터페이스만 준비)
② apply  : 미반영 체결(stateApplied=false)을 Pool·보유에 반영
           BUY: pool −= 체결가×수량 / SELL: pool += 체결가×수량 (멱등)
③ cycle  : 오늘 ≥ cycleStartDate + 14일의 첫 US 거래일이면 사이클 경계:
           pool += contribution → E(평가금) 산출 → nextV → VrCycleLog 기록
           → cycleIndex++, cycleStartDate 갱신 (cycleIndex 기반 멱등)
④ compute: order-table로 오늘의 매수·매도표 계산
⑤ submit : dryRun → simulated 기록 / LIVE → 차단(게이팅)
           전일 미체결 simulated는 sync에서 expired 처리 (당일 만료 주문이므로 매일 재계산·재제출)

blocked(일봉 미확보·크로스체크 실패 등) 시 lastRunDate를 갱신하지 않아 다음 폴에서 자동 재시도. 같은 거래일 재실행은 lastRunDate로 차단 (V4 교훈).

E(평가금) 산출 — 용도별로 분리

  • 공식의 E (사이클 경계, V 갱신용): 파생 보유수량 × 직전 US 거래일 종가(일봉) — 라오어의 "마지막 평가금 E"와 일치. 결정적(deterministic)이라 재시도에도 같은 값.
  • 대시보드 표시용 평가금: 파생 보유수량 × 당일 현재가(토스 시세) — 공식 계산에는 쓰지 않는다.
