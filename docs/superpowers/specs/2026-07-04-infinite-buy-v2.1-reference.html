무한매수법 V2.1 규칙 정리 (학습 노트)

출처: 라오어 무한매수법 카페 V2.1 공지(2021-11-13) + 요약 이미지(매도과정 요약·게으른마리 종합표·콩나물 매수도). 본 문서는 구현 전 학습·확인용이며 실제 구현은 별도 spec/plan으로 진행.

0. 핵심 개념: 전반전 / 후반전

원금소진 50%를 기준으로 나눈다. 40분할이면 하루치 = 원금/40 = 원금의 2.5%이므로:

  • 전반전 = 원금소진 < 50% (실무상 회차 < 20, 즉 round 1~19)
  • 후반전 = 원금소진 ≥ 50% (회차 ≥ 20, 즉 round 20~40)

전반전/후반전은 매수·매도 모두에 적용된다. 종목은 현재 TQQQ 위주 권장.

1. 매수 (V2.0과 동일)

하루치(원금의 2.5%)를 다음과 같이 LOC 매수로 건다. (LOC 매수 = 종가가 주문가보다 낮을 때 체결)

구간비중주문가주문유형
전반전0.5회치(원금 1.25%)내 평단가 +0% (평단)LOC 평단매수
0.5회치(원금 1.25%)내 평단가 +5%LOC 큰수매수
후반전1회치(원금 2.5%) 전액내 평단가 +0% (평단)LOC 평단매수

1회차(최초)는 평단이 없으므로 시장가로 1회치 매수해 사이클을 시작(우리 구현 관례).

큰수매수 상한 = 평단+5%. 평단+5% 초과 가격에는 매수하지 않는다. LOC 매수는 종가가 주문가보다 낮을 때만 체결되므로, 주문가를 평단×1.05로 고정하면 이 상한이 내재적으로 보장된다(코드에도 assert로 못박음).

2. 매도 (V2.1 신규 — LOC 매도를 한 칸 세분화)

매도가는 전부 내 평단가 대비. 지정가매도(=AFTER 지정가)는 장중 도달 시 체결, LOC매도는 종가가 주문가보다 높을 때 체결.

전반전 매도 (원금소진 50% 이전)

비중(전체물량 기준)매도가주문유형
25%평단 +5%LOC 매도
75%평단 +10%지정가매도

후반전 매도 (원금소진 50% 이후)

비중(전체물량 기준)매도가주문유형
25%평단 +0% (평단)LOC 매도
25%평단 +5%지정가매도
50%평단 +10%지정가매도

즉 전반→후반으로 가면 +10% 비중이 75%→50%로 줄고, 후반전엔 평단(0%) LOC 매도 25%가 신설되어 현금흐름이 개선된다.

3. 예시 (원문)

  • 전반전: SOXL 평단 $50, 100주, 15회차 → $52.5(+5%)에 LOC 25주 + $55(+10%)에 지정가 75주.
  • 후반전: SOXL 평단 $50, 200주, 30회차 → $55(+10%) 지정가 100주 + $52.5(+5%) 지정가 50주 + $50(평단) LOC 50주.

4. Q&A 요점

  • 전반전에서 LOC +5% 매도만 되고 넘어가도, 다음날 남은 물량으로 또 +5% LOC를 건다(같은 규칙 반복).
  • +5% LOC만 2~3일 연속 걸려도 그대로 진행. 3배 레버리지 특성상 곧 구간을 벗어남.
  • 후반전 0% LOC 매도만 이어진 뒤엔, 남은 물량이 전반전인지 후반전인지 보고 그에 맞는 매수로 진행.
  • 지정가 +10%만 걸리고 급락해 LOC가 안 걸리면, 남은 물량 기준으로 무매법 이어감.
  • LOC 매도로 의도보다 높은 가격(수익률 +10% 초과) 매도가 종종 발생 → 백테스트상 수익률은 V2.0과 거의 비슷.

5. 현재 우리 구현(v1 성격)과의 차이 — 중요

현재 company-map 무한매수 구현은 V2.1과 매수·매도 모두 다르다. "현재 기능 유지 + V2.1 도입"이므로 V2.1은 별도 모드로 신규 구현해야 함.

매수 큰수현재가 × (1+12%)평단 × 1.05 (평단+5%)
매수 후반전항상 평단/큰수 반반전액 평단(+0%) LOC
매도보유 전량 평단+10% 1건(LIMIT DAY)전반 2단(25%/75%)·후반 3단(25%/25%/50%), LOC+지정가 혼합
전반/후반 구분없음원금소진 50% 기준

6. Toss API 매핑 (구현 관점)

  • LOC 매수/매도 → LIMIT + timeInForce:"CLS" (검증됨)
  • 지정가매도(AFTER 지정가) → LIMIT + timeInForce:"DAY" (장중 도달 체결)
  • 후반전은 동일 종목에 동시 3개 매도 주문 필요 → 매도가능수량/동시주문 허용 여부 확인 필요.

7. 미확인 / 확인 필요 항목

  1. 쿼터손절(+쿼터손절): 사용자가 언급했으나 규칙 미제공. 손절을 4분할로 나눠 실행하는 방식으로 추정되나 정확한 트리거·비율·절차 미상 → 자료 필요.
  2. 전반/후반 경계 측정: 회차(round ≥ splits/2) 기준으로 구현할지, 실제 투입현금 기준으로 할지.
  3. 버전 병행 방식: 사이클마다 v1/v2.1 선택(cycle.version 필드)로 볼지.
  4. 동시 다중 매도 주문: Toss가 같은 보유에 2~3개 매도를 동시에 받는지(매도가능수량 분할).
  5. 가상증액/가상시드: 적용 여부(원문은 20% 이내 권장, 선택 사항).
