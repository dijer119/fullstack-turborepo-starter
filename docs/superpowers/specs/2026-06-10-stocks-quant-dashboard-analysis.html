3. 의견 요약

구현 가능성

  • 12개 항목 중 5개는 기존 데이터로 즉시 표시 가능 (기준가·시총·PBR·배당률·ROE), 2개는 근소한 추가 작업(YoY, 52주 위치)으로 가능.
  • 국채금리는 신규 소스가 필요하지만, 자주 변하지 않는 값이라 설정 화면에서 수동 입력하는 단순한 방식도 실용적임 (본인용 도구 특성상).
  • RRR·종합점수·상대순위는 출처 시트의 공식을 알아야 함 — 이미지 값으로 역산이 안 되므로 추측 구현은 위험. 공식 확보가 1순위 선결 과제.

배치 구조

이미지는 "유니버스에서 종목을 골라 보는 독립 대시보드"인 반면, 요청 위치는 종목 상세 페이지. 상세 페이지 최상단(StockDetailHeader 아래)에 대시보드 섹션을 추가하고, 종목 선택 드롭다운은 생략(URL이 이미 종목을 결정)하는 적응이 자연스러움. 순위는 StockAnalysis 보유 전 종목 또는 태그/관심종목 유니버스 기준으로 계산.

기술 스택 적합성

  • 게이지(반원형)는 이미 의존성에 있는 recharts의 RadialBarChart/PieChart로 구현 가능 — 추가 패키지 불필요.
  • 순위 계산은 Server Action에서 전 종목 1회 조회로 충분 (SQLite 로컬이라 비용 미미).
  • 다크 테마 디자인은 기존 company-map 테마 톤을 따르되, 이미지의 네온 글로우 스타일 차용 여부는 디자인 단계에서 결정.

선결 질문 (설계 전 확인 필요)

  1. RRR·종합 평균점수·국채배당 점수·상대순위의 정확한 공식/출처 (Value Sheet 양식의 정의)
  2. 순위 산정 유니버스: 분석된 전 종목 vs 관심종목(태그) vs 별도 목록
  3. 국채금리 입력 방식: 자동 수집(ECOS/스크래핑) vs 수동 설정
  4. YoY를 EPS 기준으로 맞출지, 기존 보유한 순이익/영업이익 YoY로 대체할지
