종목별 관련 링크(블로그·뉴스) 저장 기능 — 설계

앱: company-map · 작성일: 2026-05-29 · 상태: 설계 승인 대기

1. 목적 & 범위

전종목 조회(/stocks)에서 다루는 각 상장 종목에 대해, 사용자가 직접 찾은 블로그 글·뉴스 기사 URL을 종목별로 저장하고 종목 상세 페이지에서 모아 볼 수 있게 한다. URL을 붙여넣으면 서버가 OpenGraph 메타데이터(제목·출처·이미지)를 자동 추출해 채운다.

성공 기준:

  • 종목 상세 페이지에서 URL을 붙여넣어 링크를 추가하면, 제목·출처·썸네일이 자동으로 채워져 목록에 나타난다.
  • 링크의 종류(블로그/뉴스)와 한 줄 메모를 수동으로 편집·삭제할 수 있다.
  • 전종목 테이블에서 링크가 하나라도 있는 종목은 표시기(아이콘)로 구분된다.
  • 메타 추출이 실패해도 URL 자체는 항상 안전하게 저장된다.

비범위 (YAGNI)

  • 네이버 뉴스 검색 API 등 자동 기사 수집 — 이번 범위 아님(수동 저장만).
  • 전종목 테이블의 링크 필터 토글 — 사용자 결정으로 제외(표시기만).
  • 백그라운드 워커를 통한 비동기 메타 추출 — 1인용 도구엔 과함.

2. 접근 방식

저장 시 동기 best-effort fetch (접근법 A)를 채택한다. server action이 URL을 받는 즉시 fetch + cheerio로 OG 메타를 파싱(5초 타임아웃)하고, 성공하면 채우고 실패/타임아웃이면 URL만 저장한다. 기존 setMemo류 server action 패턴과 동일하며 새 의존성이 없다(cheerio 기존 사용).

3. 데이터 모델

prisma/schema.prisma에 신규 모델 StockLink를 추가하고, StockMaster에 links StockLink[] relation을 추가한다.

model StockLink {
  id        String   @id @default(uuid())
  code      String
  url       String
  title     String?                      // og:title (자동 추출)
  siteName  String?  @map("site_name")   // og:site_name (자동 추출)
  imageUrl  String?  @map("image_url")   // og:image (자동 추출)
  kind      String   @default("blog")    // "blog" | "news"
  memo      String?                      // 수동 한 줄 메모
  createdAt DateTime @default(now()) @map("created_at")

  master    StockMaster @relation(fields: [code], references: [code], onDelete: Cascade)

  @@unique([code, url])                  // 같은 종목 내 URL 중복 방지 (idempotent)
  @@index([code, createdAt])
  @@map("stock_links")
}

StockMaster 모델에 한 줄 추가:

  links  StockLink[]

마이그레이션: yarn workspace company-map prisma migrate dev --name add_stock_links

og-meta 유틸 상세

  • parseOgMeta(html, url) → { title, siteName, imageUrl }. cheerio로 og:title/og:image/og:site_name 우선, 없으면 <title>·twitter:* 폴백. 순수 함수 — 테스트 대상.
  • fetchOgMeta(url) → AbortController 5초 타임아웃으로 HTML 받아 parseOgMeta 호출. 실패 시 빈 메타 {} 반환(throw 금지).
  • inferKind(url) → 호스트 휴리스틱. news.naver.com, n.news.naver.com, 주요 언론사 도메인 → "news", 그 외 → "blog". 순수 함수 — 테스트 대상.

5. 데이터 흐름

  1. 상세 페이지 진입: page.tsx가 db.stockLink.findMany({ where: { code }, orderBy: { createdAt: "desc" } })로 링크 로드 → RelatedLinksCard에 prop 전달.
  2. 링크 추가: 카드에서 URL 입력 → addLink(code, url) 호출 → 서버가 URL 검증 → fetchOgMeta + inferKind → db.stockLink.create(중복이면 기존 반환) → 생성 row 반환 → 클라이언트가 목록 상단에 추가.
  3. 편집/삭제: updateLink(id, {kind, memo}) / deleteLink(id) → 클라이언트 상태 갱신.
  4. 전종목 테이블: getStocksExplorer가 hasLinks를 집계(예: _count 또는 존재 여부 join) → 클라이언트가 표시기 렌더.

6. 에러 처리

상황처리
코드가 6자리 숫자 아님저장 안 하고 에러 객체 반환 (memos.ts와 동일 가드)
URL이 http(s) 아님저장 안 하고 에러 반환
fetch 실패 · 타임아웃 · 비-HTML 응답URL + inferKind 결과만 저장, 메타는 null. 절대 throw 안 함
(code, url) 중복기존 row 반환 (idempotent, @@unique로 보장)
cheerio 파싱 중 누락 태그안전 폴백 체인으로 가능한 필드만 채움

7. 테스트 (vitest, 앱 기존 패턴)

  • src/lib/links/og-meta.test.ts
    • parseOgMeta: OG 태그 완비 HTML → 전 필드 추출 / OG 누락 HTML → <title>·twitter 폴백 / 빈 HTML → 모두 null. (네트워크 없이 HTML 문자열로)
    • inferKind: 네이버 뉴스 도메인 → "news", 티스토리·네이버 블로그 → "blog".

네트워크에 의존하는 fetchOgMeta는 파싱 로직을 parseOgMeta로 분리해 테스트 가능하게 한다.
