지분 공시 상세 — Design Spec

Date: 2026-05-27

Scope: 지분 공시 카드에 보고자·직위·거래수량 등 상세 정보 표시. DART elestock / majorstock 상세 API 호출 + payload 확장. 자기주식·기타는 기존 제목+링크 fallback 유지.

Goals

  • 임원·주요주주 카드를 이미지처럼 상세 표시 (보고자, 직위, 보고사유, 변동일, 거래수량±, 가격, 금액, 거래후 보유).
  • 주식대량보유 카드 상세 표시 (보고자, 보유주식수, 보유비율, 변동비율, 보고사유).
  • 기존 fetch 흐름에 자연스럽게 통합: fetchOwnership 안에서 type별 분기.

Non-Goals (이번 Phase)

  • 자기주식 취득/처분 결정 상세 — DART API 종류 복잡 (취득결정·처분결정·신탁·소각 등 다수). Phase 3로 이연. 이번에는 기존 제목+DART 링크.
  • 임원 등기/미등기 구분, 주요주주 여부 등의 세분 필드 — payload에 position 1개 필드로 통합.
  • 지분 fetch 자동 갱신 (lazy seed 외) — 사용자가 "공시 갱신" 버튼으로 명시 갱신.

Payload 확장

src/lib/dart/disclosure-payloads.ts:

export interface ElestockDetail {
  type: "elestock";
  filer: string | null;            // 보고자
  position: string | null;         // 직위 (예: 상무, 등기임원, 주요주주)
  reason: string | null;           // 보고사유 (예: "장내매수(+)", "장내매도(-)", "증여")
  changeDate: string | null;       // 변동일 YYYY-MM-DD
  tradeQuantity: number | null;    // 거래수량 (±, 매수=+, 매도=-)
  tradePrice: number | null;       // 단가 (원)
  tradeAmount: number | null;      // 금액 (원)
  afterShares: number | null;      // 거래후 보유주식수
}

export interface MajorstockDetail {
  type: "majorstock";
  filer: string | null;            // 보고자
  ownedShares: number | null;      // 보유주식수
  ownedRatio: number | null;       // 보유비율 (%)
  changeRatio: number | null;      // 직전 보고 대비 변동 (%p)
  reason: string | null;           // 보고사유
}

export type OwnershipDetail = ElestockDetail | MajorstockDetail;

export interface OwnershipPayload {
  pblntfDetailTy?: string;
  reportType: "주식대량보유" | "임원·주요주주" | "자기주식" | "기타";
  detail?: OwnershipDetail | null;   // 자기주식·기타는 null
}

DART API

elestock.json (임원·주요주주특정증권등소유상황보고서)

엔드포인트: https://opendart.fss.or.kr/api/elestock.json

파라미터:

  • corp_code
  • bgn_de, end_de

주의: 정확한 응답 필드명은 OpenDART 문서 확인. 추정 필드 (확인 후 조정):

  • repror: 보고자 이름
  • isu_exctv_ofcps / isu_exctv_rgist_at: 직위·등기여부
  • stkrt_irds_resn: 변동 사유
  • trde_dd: 거래일
  • trde_qy: 거래수량 (음수 가능)
  • unit_pic / trde_uprc: 단가
  • trde_amt: 거래금액
  • posesn_stock_co_after: 거래후 보유

응답 list에는 종목의 모든 임원·주요주주 보고가 들어옴. rcept_no로 우리 disclosures의 rcpNo와 매칭.

majorstock.json (주식등의대량보유상황보고서)

엔드포인트: https://opendart.fss.or.kr/api/majorstock.json

응답 추정 필드:

  • repror: 보고자
  • stkqy: 보유주식수
  • stkrt: 보유비율
  • stkrt_irds: 변동비율
  • report_rs: 보고사유 (단순투자/일반투자/경영참여 등)
  • rcept_no

호출 패턴

두 API는 종목당 1회 호출 (corp_code + bgn_de/end_de 4년치). 응답을 rcpNo로 indexing하는 Map 생성. fetchOwnership의 row 루프에서 reportType에 따라 Map lookup.

API 부하: 종목당 list.json(D) 1회 + elestock.json 1회 + majorstock.json 1회 = +2 호출 (이전 추정 16회보다 훨씬 적음 — 개별 row마다가 아닌 종목당 batch).

검증 기준

  • tsc EXIT=0. 마이그레이션 불필요.
  • "공시 갱신" 누름 → 종목당 추가 ≈ 2~3초.
  • 삼성전자(005930) 페이지에서 임원·주요주주 카드에 보고자/직위/거래수량 등 노출.
  • 주식대량보유 row가 존재하는 종목(예: VIP 보유 종목 중 일부)에서 majorstock 카드 노출.
  • 자기주식·기타 row는 기존과 동일 (제목+링크).
  • 상세 API 실패 시 detail=null로 fallback (카드는 제목+링크 형식으로 보임).
  • 거래수량 +/- 부호와 색상 (녹/빨).

Out of Scope / Open Questions

  • elestock/majorstock 응답의 정확한 필드명은 구현 시 1회 시범 호출로 확인 후 정규식·필드 매핑 조정. Phase 2 alotMatter 패턴과 동일한 위험.
  • 임원 등기/미등기, 6%룰 추가/특수관계인 등 세분 — 표시 안 함, 1개 position 필드로 통합.
  • 거래후 보유주식수 = 0인 row (전량 매도) 색상 처리 — null 처리와 동일하게 "0주" 표시. 별도 강조 없음.
