공시 History Phase 2 — 배당 + 지분

Date: 2026-05-27

Scope: 종목 상세 페이지 timeline에 "배당", "지분" 카테고리 추가. 기존 "실적" 그대로 유지. Phase 1의 Disclosure 테이블 재활용.

Goals

  • 종목 페이지에 카테고리 3개 동시 노출: 실적 / 배당 / 지분.
  • 배당 카드: 주당 배당금 · 시가배당률 · 배당기준일 · 공시일 (DART 상세 API).
  • 지분 카드: 보고서 종류(주식대량보유·임원·주요주주 등) · 공시일. (list.json만 사용, 상세 fetch 없음.)
  • fetchDisclosuresForStock 한 호출로 세 카테고리 모두 갱신.

Non-Goals (Phase 2)

  • 지분 변동 상세 (보유 비율·증감·사유) — Phase 3에서 stkrtChg API 추가 시 다룸. 현재는 list.json의 report_nm만.
  • 합병·분할·증자 등 다른 주요사항 카테고리 — Phase 3.
  • VipHolding 데이터 Disclosure로 마이그레이션 — 기존 VipHolding 그대로 유지 (행 확장 표시 그대로).

Data Model

기존 Disclosure 테이블 변경 없음. category에 "지분", "배당" 추가만.

OwnershipPayload (지분 카드 payload)

interface OwnershipPayload {
  pblntfDetailTy?: string;    // DART 세부 코드 (있을 때만)
  reportType: string;         // "주식대량보유" | "임원·주요주주" | "자기주식" | "기타"
  // dartUrl은 rcpNo로 https://dart.fss.or.kr/dsaf001/main.do?rcpNo={rcpNo} 생성. payload에 별도 저장 안 함.
}

DividendPayload (배당 카드 payload)

interface DividendPayload {
  divPerShare: number | null;       // 주당 배당금 (원)
  dividendYieldPct: number | null;  // 시가배당률 (%)
  recordDate: string | null;        // 배당기준일 "YYYY-MM-DD"
  dividendType: "현금" | "주식" | "기타";
}

DART API

지분 — list.json(pblntf_ty=D)만 사용

응답 row의 report_nm으로 reportType 분류:

  • /주식등의대량보유상황보고서/ → "주식대량보유"
  • /임원·주요주주특정증권등소유상황보고서/ → "임원·주요주주"
  • /자기주식/ → "자기주식"
  • 그 외 → "기타"

배당 — list.json(pblntf_ty=B) 필터 + 현금배당 상세

1차 list.json: pblntf_ty=B. row 중 report_nm에 "현금·현물배당결정" / "주식배당결정" 포함된 것만 채택.

2차 상세 API: DART 단일회사 주요계정 API군에 "현금·현물배당 결정" 별도 endpoint 존재. 정확한 URL은 OpenDART 문서 확인 (예: https://opendart.fss.or.kr/api/cashDvdndPaymntDcsn.json). 응답 파싱 후 DividendPayload 추출. 실패 시 payload=null로 저장 (카드에 "—" 표시).

API 부하 추정

  • list.json: 종목당 3회 (A, B, D).
  • fnlttSinglAcnt.json: 실적 카테고리 16건 × 1회 = 16 (Phase 1 그대로).
  • cashDvdndPaymntDcsn.json: 배당 row 수만큼 (보통 4년 × 1~2회 = 4~8).
  • 총 종목당 ≈ 25 API calls. 100ms 딜레이 시 ≈ 2.5초.

검증 기준

  • tsc EXIT=0.
  • 마이그레이션 불필요 (스키마 변경 없음).
  • "공시 갱신" 버튼 한 번 누름 → 종목당 ≈ 25 API calls (수 초).
  • 삼성전자(005930) 페이지: 실적·배당·지분 3개 섹션 모두 카드 노출.
  • 배당이 없는 종목(예: 카카오뱅크 — 배당이 있다면 다른 종목 선택): 배당 섹션 "공시 없음".
  • 지분 카드의 DART 원본 링크 클릭 시 새 탭에서 dart.fss.or.kr 원본 보고서 열림.
  • 한 카테고리 fetch 실패 시 다른 카테고리는 정상 표시 (격리 검증).

Out of Scope / Open Questions

  • DART 배당 상세 API의 정확한 endpoint와 응답 스키마는 구현 시 OpenDART 문서로 확인. 응답 필드명 변화 가능성 있음.
  • 주식배당 (현금 아닌)은 별도 API 또는 별도 카드 처리 필요할 수 있음 — Phase 2에서는 현금배당만 정확히, 주식배당은 dividendType: "주식"으로 표시하되 금액은 null로 둠.
  • 지분 카드의 "보고자명" 추출 — list.json의 응답에는 안 들어있음. 상세 fetch 없는 한 표시 불가. Phase 3에서 stkrtChg/elestock API 도입 시 추가.
