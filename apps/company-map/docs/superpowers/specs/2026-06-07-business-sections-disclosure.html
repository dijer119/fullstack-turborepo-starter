종목 상세에 사업개요·매출·수주상황 추가 — Design Spec

Date: 2026-06-07

Scope: 종목 상세(/stocks/[code])의 "공시 갱신" 시 DART 정기보고서 본문 "II. 사업의 내용"에서 사업개요·매출·수주상황을 추출해 새 섹션으로 표시한다. 사업개요·매출은 최신 보고서 1개, 수주상황은 분기별로 누적한다.

Goals

  • 종목 상세에서 사업개요(서술), 매출(매출실적 표), 수주상황(수주잔고 표)을 공시 타임라인에 추가.
  • 수주상황은 정기보고서(분기/반기/사업)마다 스냅샷을 누적해 수주잔고 변화를 시점별로 확인.
  • 회사마다 표 구조가 달라도 견고하도록, 보고서 표/본문을 정제(sanitize)해 그대로 렌더링.

Non-Goals

  • 매출·수주 표를 고정 컬럼으로 구조화 파싱 — 회사별 형식 차이로 누락 위험. 원문 표 그대로 렌더.
  • 사업개요·매출의 분기별 누적 — 최신 1개만(덮어쓰기).
  • 비정기공시·해외법인 보고서 — 정기보고서(pblntfTy A)만.

Decisions

항목결정
사업개요 / 매출최신 정기보고서 1개 (덮어쓰기)
수주상황분기별 누적 (정기보고서별 스냅샷)
추출·표시보고서 섹션 HTML을 sanitize 후 그대로 렌더(표/문단)
섹션 위치보고서 main.do의 treeData(목차) 파싱으로 노드 찾기
저장기존 Disclosure 테이블 + 합성 rcpNo 키
트리거기존 fetchDisclosuresForStock("공시 갱신")에 통합

데이터 소스 / 섹션 위치 (treeData)

정기보고서 https://dart.fss.or.kr/dsaf001/main.do?rcpNo=... 페이지에는 목차가 treeData JS로 들어있고, 각 노드가 제목→문서 파라미터를 매핑한다(실측 확인). 노드 블록 예:

var node2 = {};
node2['text']   = "4. 매출 및 수주상황";
node2['dcmNo']  = "11386164";
node2['eleId']  = "13";
node2['offset'] = "60425";
node2['length'] = "26562";
node2['dtd']    = "dart4.xsd";

제목 정규화 매칭으로 노드를 찾는다:

  • 사업개요 ← 제목에 "사업의 개요" 포함하는 노드
  • 매출 + 수주상황 ← 제목에 "매출" 그리고 "수주" 포함하는 노드("4. 매출 및 수주상황"). 이 한 섹션 안에 매출실적·판매경로·수주상황 소제목과 표가 함께 있음(005930 실측: 매출실적 5회·판매경로 4회·수주상황 2회·표 20개).
OpenDART API 키 한도와 무관: dsaf001/main.do·report/viewer.do는 공개 문서 뷰어다. 단 본문은 회사마다 EUC-KR/UTF-8이 섞여 기존 dart-viewer.ts의 charset 처리를 재사용한다.

추출 (기존 dart-viewer.ts 재사용)

  • fetchViewerHtmlByParams(rcpNo, dcmNo, eleId, offset, length, dtd)로 섹션 HTML 취득(기존 fetchViewerHtml은 main.do에서 첫 viewDoc만 씀 → 파라미터 직접 지정 버전 추가).
  • 사업개요: 섹션 HTML을 sanitize한 문단/텍스트.
  • 매출: "매출 및 수주상황" 섹션에서 "수주상황" 소제목 이전의 표(매출실적)들.
  • 수주상황: 같은 섹션에서 "수주상황" 소제목 이후의 표들.
  • 분할은 소제목 텍스트 위치 기반 휴리스틱. 소제목/표 미발견 시 빈 결과 → 해당 카드 미생성.

sanitize: <script>/<style>/<link>/<meta> 제거, on* 이벤트 속성·javascript: 제거. 표/문단/텍스트는 보존. (company-map에 없는 유틸이므로 src/lib/dart/에 작은 sanitize 함수 신설; blog-collection의 접근 차용.)

저장 (Disclosure + 합성 키)

새 category 3개. 기존 배당의 alotMatter-{code}-... 합성키 패턴 차용(같은 보고서 rcpNo가 여러 섹션에 쓰여 충돌하므로 합성키 필요).

categoryrcpNo(합성)범위payload
사업개요bizoverview-{code}최신 1개(덮어쓰기){ html, sourceRcpNo, reportNm }
매출sales-{code}최신 1개{ html, sourceRcpNo, reportNm }
수주orders-{code}-{bsnsYear}-{reprtCode}보고서별 누적{ html, sourceRcpNo, period }

DisclosureCategory 타입(disclosure-payloads.ts)에 "사업개요" | "매출" | "수주" 추가. payload 타입 SectionPayload { html: string; sourceRcpNo: string; reportNm?: string; period?: string } 신설.

수집 (src/actions/disclosures.ts)

  • 새 함수 fetchBusinessSections(code, corpCode)를 추가하고 fetchDisclosuresForStock의 수집 함수 목록에 포함.
  • 정기보고서 목록(pblntfTy A, 기존 4년 윈도우)을 최신순 정렬.
  • 최신 1개: main.do → treeData 파싱 → 사업개요·매출/수주 노드 → viewer.do → 사업개요·매출 저장(덮어쓰기), 수주도 저장.
  • 수주 분기별 누적: 그 외 최근 보고서들(최대 8개)에 대해 이미 저장된 합성키는 dedup skip, 신규 보고서만 main.do+viewer.do로 수주상황 추출·저장.
  • 종목당 호출 수 bound: 보고서 8개 × (main.do 1 + viewer.do 1~2). 개별 보고서 실패는 무시하고 계속.

UI (DisclosureTimeline + SectionNav + 카드)

  • SectionNav에 사업개요 · 매출 · 수주 항목 추가(점 색상 신규 배정).
  • DisclosureTimeline에 3개 Section 추가. 카드는 sanitize된 html을 dangerouslySetInnerHTML로 렌더(표 스타일 wrapper: 테두리·헤더배경·가로 스크롤).
  • 사업개요·매출: 카드 1개(기준 보고서 표기 + DART 원본 링크).
  • 수주: 분기별 카드 여러 개(기간 period 표기, 최신순).
  • page.tsx에서 category별 필터링해 props 전달(기존 earnings/dividends/… 패턴 동일).

에러 / 엣지

  • treeData에 해당 섹션 없음(금융사 등 수주상황 없음) → 해당 카드 미생성.
  • 제목 변형(번호·띄어쓰기) → 공백 제거 후 부분일치 매칭.
  • viewer.do charset(EUC-KR/UTF-8) → 기존 dart-viewer charset 처리 재사용.
  • 개별 섹션/보고서 실패가 다른 공시(실적·배당 등) 수집을 막지 않음(try/catch).

테스트 (Vitest)

  • parseTreeData(mainHtml) → 제목→{dcmNo,eleId,offset,length,dtd} 매핑 (실제 main.do 픽스처).
  • 섹션 분할: "수주상황" 소제목 기준 매출/수주 표 분리 (실제 섹션 HTML 픽스처).
  • sanitize: script/이벤트핸들러 제거, 표 보존.
