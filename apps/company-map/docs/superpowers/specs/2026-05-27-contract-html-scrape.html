계약 공시 HTML 스크래핑

Date: 2026-05-27

Scope: 단일판매·공급계약체결 공시의 상세 정보를 DART 본문 HTML 스크래핑으로 확보. ContractCard에 이미지처럼 계약일·기간·내용·상대방·지역·금액·매출액 대비 노출.

Goals

  • rcpNo로 DART 본문 HTML 가져와 정규식으로 항목 추출.
  • 기존 메타데이터-only 저장을 상세 payload 저장으로 업그레이드.
  • 추출 실패 시 graceful fallback (null payload, 카드는 제목+링크).

Non-Goals

  • cheerio·jsdom 등 외부 의존성 추가 — 정규식만 사용 (가벼움).
  • 다른 종류 공시 스크래핑 (자사주 결정·합병 등) — 후속.
  • 본문 첨부파일(XBRL ZIP) 다운로드 — viewer.do HTML만 사용.

DART HTML 구조 분석

1단계: dsaf001/main.do

URL: https://dart.fss.or.kr/dsaf001/main.do?rcpNo={rcpNo}

응답 HTML에 JS:

node1['text'] = '...';
node2[1] = {text: '...', id: '...'};
viewDoc('20260317800123', '7654321', '0', '0', '12345', 'dart3.xsd');

또는 페이지 안에 <a onclick="openPdfDownload(...)"> 같은 요소. 핵심 추출 대상: dcmNo (두 번째 인자). 그 외 eleId·offset·length·dtd는 viewer URL 구성에 필요.

2단계: report/viewer.do

URL: https://dart.fss.or.kr/report/viewer.do?rcpNo={rcpNo}&dcmNo={dcmNo}&eleId={eleId}&offset={offset}&length={length}&dtd={dtd}

응답 HTML 안에 <table>로 항목·값 쌍. 단일판매·공급계약 보고서 표준 양식:

<tr><td>1. 판매·공급계약 내용</td><td>Valve remote control system외 판매계약</td></tr>
<tr><td>2. 계약내역</td><td>...</td></tr>
<tr><td>  - 계약금액(원)</td><td>18,281,120,000</td></tr>
<tr><td>  - 최근 매출액(원)</td><td>104,200,000,000</td></tr>
<tr><td>  - 매출액 대비(%)</td><td>17.54</td></tr>
<tr><td>3. 계약상대방</td><td>SINOKING ENGINEERING CO.,LIMITED(HENGLI SHIPBUILDING)</td></tr>
<tr><td>5. 판매·공급지역</td><td>중국</td></tr>
<tr><td>6. 계약기간</td><td>2026-03-17 ~ 2029-03-30</td></tr>
<tr><td>9. 계약(이사회결의)일자</td><td>2026-03-17</td></tr>

표 마크업은 변형 가능. 정규식은 라벨 패턴 우선 + 값 후속 <td> 안 텍스트 추출.

구현 모듈

src/lib/dart/contract-detail.ts (신규)

1. fetchDartPage(url): Promise<string | null> — User-Agent 헤더 + 30초 timeout + retry 3.

2. extractDcmNo(mainHtml): string | null — main.do HTML에서 viewDoc('', '', ...) 정규식 매칭.

3. parseContractBody(viewerHtml): Partial<ContractPayload> — 표 라벨·값 추출.

4. fetchContractDetail(rcpNo): Promise<ContractPayload | null> — 위 3개 조합.

API 부하: 종목당 list 1회 + 계약 row 수만큼 본문 fetch. 보통 4년 5~16건이라 수 초.

검증 기준

  • tsc EXIT=0.
  • 계약 공시 있는 종목(예: 한화오션 042660, HD현대중공업 등)에서 "공시 갱신" → 계약 카드에 계약일/기간/금액/상대방/지역 노출.
  • 일부 필드만 매칭되더라도 카드는 정상 렌더 (NULL 행 자동 숨김 — 이미 ContractCard에 적용됨).
  • HTML 구조 깨질 가능성 → 추출 실패해도 카드는 제목+링크 fallback (현재 메타데이터-only 동작과 동일).

위험 / Open Questions

  • DART 페이지 구조 변경 시 정규식 fragile. 추출 실패 모니터링은 없음 (콘솔 경고만).
  • 일부 보고서가 [기재정정] 등 형식이 다른 경우 — 항목 위치 변동 가능.
  • "계약내용"이 매우 긴 경우 카드에서 line break 처리 — ContractCard의 Row 컴포넌트가 자동 wrap.
  • main.do 응답이 charset=euc-kr 일 가능성. Content-Type 헤더 확인하고 필요시 디코딩 (Node fetch는 기본 utf-8 가정 — 깨질 수 있음). 구현 단계에서 확인 + iconv-lite 같은 의존성 회피 위해 utf-8 가정 우선, 깨지면 별도 처리.
