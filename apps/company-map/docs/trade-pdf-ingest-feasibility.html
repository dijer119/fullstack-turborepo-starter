관세청 旬報 PDF 데이터 적재 가능성 검토

대상 파일: korea.kr download.do?fileId=198487132&tblKey=GMN
실제 내용: 관세청 「2026년 6월 1일~6월 10일 수출입 현황 [잠정치]」 보도자료 (배포 2026.6.11)
검토일: 2026-06-16

결론 한 줄: PDF 다운로드 & 텍스트 추출 자체는 가능하지만, 현재 trade_stats 테이블에 그대로 넣는 것은 권장하지 않음. PDF가 가진 데이터의 입도(granularity)·분류체계·집계방식이 앱 모델과 근본적으로 다르고, 세부 품목/국가 수치는 차트 이미지라 텍스트로 추출되지 않는다.

1. PDF에서 실제로 추출되는 것

pdftotext -layout로 추출 검증 완료. 텍스트 레이어가 살아있어 OCR은 불필요하나, 추출 가능 범위가 제한적이다.

항목추출 가능?형태
총괄 수출/수입/무역수지 (국가 전체 합계)가능총괄 표 텍스트 (당월·연간누계·전월 금액)
주요품목 증감률 (반도체 205.8%, 석유제품 68.7% …)일부본문 산문 속 % 만. 금액·비중은 차트 이미지
주요국가 증감률 (중국 101.4%, 미국 54.4% …)일부본문 산문 속 % 만
품목별 금액·비중 표불가"주요품목 수출현황(억달러, 비중%)" 아래가 전부 차트 그래픽 — 텍스트 0
국가별 금액 표불가마찬가지로 차트 이미지

추출된 총괄 표 예시(2026년 당월 = 6.1~10일, 단위 백만 달러):

수출  28,635  (전년동기 23,352 대비 +85.9%)   연간누계 423,096
수입  23,352(전월)/실제 ...  (+35.6%)          연간누계 315,642
무역수지  5,282                                연간누계 107,454

주의: 컬럼이 △·괄호·줄바꿈으로 흩어져 파싱 로직이 까다롭고, 월마다 레이아웃이 흔들릴 수 있음.

2. 앱 데이터 모델과의 불일치 (핵심 장애)

2-1. 입도(granularity) 불일치

현재 trade_stats 모델은 HS코드 × 국가 × 월 단위로 금액을 저장한다.

@@unique([category, yearMonth, hsCode, countryCd])
expDlr / impDlr / expWgt / impWgt / balPayments  (모두 행 단위 금액)

그러나 PDF 旬報는 국가 전체 총괄 + 품목군 단위 증감률만 텍스트로 제공한다. HS코드별·국가별 금액(= 이 테이블의 한 행을 채우는 데 필요한 값)은 PDF에 텍스트로 존재하지 않는다.

2-2. 분류체계 불일치

앱 카테고리(semiconductor, cosmetics, power-equipment, transformer, defense, shipbuilding, battery, ess, nuclear)는 HS코드 리스트로 정의된다. PDF는 관세청 "성질별 분류체계"(반도체·석유제품·승용차·원유 등)로 집계되며, 두 분류는 1:1로 매핑되지 않는다. (PDF 각주: "주요품목은 관세청 성질별 분류체계 기준으로 집계")

2-3. 집계방식·기간 불일치

구분앱 사용 소스 (nitemtrade API)이 PDF (旬報)
기간월 전체 (확정/잠정 월간)1~10일 등 10일 단위 단기
성격품목·국가별 월간 통계국가 총괄 속보 잠정치
확정 시점익월 중순경2027.2월 연간확정까지 수시 정정
두 데이터를 같은 trade_stats에 섞으면 월간 합계·비중 계산이 오염된다 (서로 다른 기간·방법론).

3. 선택지

안내용평가
A. 구조화 소스 우선 탐색 권장 관세청/data.go.kr에 旬報(1~10·1~20일 수출입현황)를 제공하는 API·CSV·OpenAPI가 있는지 먼저 확인. 있으면 PDF 파싱 불필요. 최선 — 견고함. PDF 레이아웃 깨짐 리스크 0
B. 총괄만 별도 모델로 적재 PDF에서 국가 총괄 수출/수입/무역수지만 추출해 신규 TradeProvisional 모델에 저장. "진행 중인 월의 잠정 속보" 배너/카드로 표시. trade_stats는 건드리지 않음. 제한적 가능 — 품목/국가 세부는 못 넣음. 새 의존성(pdf 파서) 필요
C. trade_stats에 직접 적재 PDF 데이터를 기존 테이블에 삽입. 불가/비권장 — 2장의 3가지 불일치로 데이터 오염
D. OCR/이미지 파싱으로 세부 표 복원 차트 이미지에서 품목/국가 금액을 OCR. 비권장 — 신뢰도 낮고 유지보수 부담 큼

4. 권장 경로

  1. 먼저 구조화 소스 확인(안 A): data.go.kr에서 "수출입 현황 / 旬報 / 10일" 관련 OpenAPI 유무 조사. 있으면 그걸로 진행.
  2. 없다면 안 B: PDF에서 총괄 3개 지표만 뽑아 별도 TradeProvisional에 저장하고, /trade에 "6월 잠정(1~10일)" 보조 표시. 품목/국가 세부는 "확정통계 발표 후 제공"으로 안내.
  3. 구현 시 신규 런타임 의존성(예: unpdf / pdf-parse) 1개 추가 필요 — 현재 프로젝트엔 Adobe PDF 파서가 없음 (기존 "etf_pdf"는 KRX의 Portfolio Deposit File로, 동명이의).

5. 검증에 사용한 명령

curl -sL "https://www.korea.kr/common/download.do?fileId=198487132&tblKey=GMN" -o korea_trade.pdf
file korea_trade.pdf          # PDF 1.6, 텍스트 추출 가능
pdftotext -layout korea_trade.pdf korea_trade.txt
# → 총괄 표 텍스트 OK / 품목·국가 표는 차트 이미지라 텍스트 없음
