대상: fullstack-turborepo-starter/apps/company-map + 공유 키 소비자 · 조사일 2026-05-30

일일 한도(기본 20,000건/일)를 소비하는 것은 crtfc_key를 쓰는 OpenDART API(opendart.fss.or.kr/api/*) 호출뿐입니다. 공시 원문 뷰어(dart.fss.or.kr/dsaf001, /report/viewer.do) 스크래핑은 키를 쓰지 않아 한도와 무관합니다.
중요: company-map 앱과 DART MCP 서버가 같은 키(51cdca36…a9f4)를 공유해 동일한 2만 건 풀을 나눠 씁니다.

3. DART MCP 서버 (같은 키 공유)

dart.log 확인 결과 동일 키(51cdca36…)로 corpCode.xml, fnlttSinglAcntAll.json(전체 재무제표 — 무거움), list.json을 호출합니다. 이 세션에서 쓰는 다음 도구들이 company-map과 같은 2만 건 풀을 소비합니다:

  • mcp__DART__get_disclosurelist / get_disclosure → list.json 등
  • mcp__DART__get_complete_financial_statements / get_xbrl_financial_statements → fnlttSinglAcntAll.json (호출당 무거움)
  • mcp__DART__get_single_company_key_indicators, 각종 채권/증자 발행 조회 도구
  • mcp__DART__get_corpcode / update_corplist → corpCode.xml

→ 모닝브리프·산업분석 보고서를 만들 때 이 MCP 도구를 다수 호출하면 한도가 빠르게 소진됩니다(오늘 020의 유력 원인).

4. 한도와 무관 (참고)

  • dart.fss.or.kr 공시 원문 뷰어 스크래핑: contract-detail.ts(계약 상세), OwnershipCard/ContractCard의 링크, vip-holdings.ts의 dartUrl — crtfc_key 미사용이라 일일한도와 무관.

5. 한도 절약 팁

  • 가장 큰 소비처는 worker NCAV 전종목 배치와 MCP 전체 재무제표 조회. 빈도/범위를 줄이거나 결과를 캐싱(이미 DB에 저장하는 구조이므로 재호출 회피).
  • "공시 갱신"은 이미 DB에 있으면 재fetch하지 않도록 가드돼 있음(상세 페이지). 강제 갱신 남발 자제.
  • company-map과 MCP에 키를 분리하면 한쪽이 다른 쪽 한도를 잠식하지 않음.
  • 필요 시 OpenDART에 사용한도 증량 신청.
